# Contributing to Quellog

Thank you for considering contributing to Quellog! We welcome contributions that
improve the functionality, documentation, or performance of this tool. Below are
some guidelines to help you get started.

## How to Contribute

1. **Fork the Repository:**  
   Create your own fork of the project on GitHub.

2. **Create a Feature Branch:**  
   Use a descriptive name for your branch (e.g., `feature/improve-parsing` or
   `bugfix/fix-date-filter`).

3. **Write Your Code:**  
   - Follow the existing code style and conventions.
   - Add tests where applicable.
   - Ensure your code is well documented.

4. **Commit Changes:**  
   Write clear, concise commit messages describing your changes.

5. **Submit a Pull Request:**  
   Create a pull request to the main repository. Please include a description of
   the changes, why they are necessary, and any relevant issue references.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub.
Include as much detail as possible (steps to reproduce, expected vs. actual
behavior, environment, etc.).

## Code of Conduct

Please review our [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for the guidelines
governing behavior in our community.

## Additional Notes

- **Testing:**  
  Ensure that your changes pass all tests before submitting a pull request. Run
  tests with:
  ```sh
  go test ./...
  ```
- **Documentation:**  
  Update documentation as needed to reflect your changes.

Thank you for helping make Quellog a better project!
